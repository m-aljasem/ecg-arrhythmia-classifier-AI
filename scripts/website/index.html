Model Architectures

Three specialized models designed for different use cases and performance requirements

Model 01

75%

Metadata-Only

  • Uses patient metadata only
  • Age, gender, baseline features
  • Lightweight & fast inference
  • Best for quick screening

Architecture: Dense layers (128 → 64 → 32 units) with BatchNorm and Dropout

Model 02

89%

Combined Model

  • Integrates ECG signals + metadata
  • 1D CNN for signal processing
  • Highest accuracy achieved
  • Best for clinical accuracy

Architecture: Dual-branch: Conv1D layers for ECG + Dense for metadata, concatenated embeddings

Model 03

87%

Alternative Combined

  • Alternative architecture design
  • Different hyperparameters
  • Balanced performance
  • Good generalization

Architecture: Modified Conv1D structure with different kernel sizes and pooling strategies

Diagnostic Classes

Our model can identify five major cardiac conditions

✓

NORM - Normal ECG

Healthy cardiac rhythm with no abnormalities detected. Baseline reference for comparison.

⚠️

MI - Myocardial Infarction

Heart attack indicators including ST elevation and abnormal Q waves in ECG signals.

📉

STTC - ST/T Changes

Abnormalities in ST segment and T wave morphology indicating ischemia or other conditions.

⚡

CD - Conduction Disturbance

Irregular electrical conduction patterns including bundle branch blocks and AV blocks.

💓

HYP - Hypertrophy

Enlarged heart chambers detected through voltage criteria and QRS morphology changes.
